# Firstframe V1 — Complete Setup Guide

## ⚡ Quick Start (5 minutes)

### Step 1: Create a Firebase Project

1. Go to [https://console.firebase.google.com](https://console.firebase.google.com)
2. Click **"Create a project"** → Name it `firstframe` → Continue
3. Disable Google Analytics (not needed for V1) → Create Project

### Step 2: Enable Firebase Services

#### Authentication
1. In Firebase Console → **Build** → **Authentication** → **Get Started**
2. Go to **Sign-in method** tab → Enable **Email/Password**

#### Firestore Database
1. **Build** → **Firestore Database** → **Create database**
2. Choose **Start in test mode** (for development)
3. Select a Cloud Firestore location close to India (e.g. `asia-south1`)

#### Storage
1. **Build** → **Storage** → **Get Started**
2. Start in **test mode**

### Step 3: Get Your Firebase Config

1. In Firebase Console → ⚙️ **Project Settings** (gear icon top-left)
2. Scroll down to **"Your apps"** → Click **Web** (`</>` icon)
3. Register app name: `firstframe-web` → Register
4. Copy the `firebaseConfig` object — you'll need these values

### Step 4: Get Service Account Key (for server-side)

1. In Firebase Console → ⚙️ **Project Settings** → **Service accounts** tab
2. Click **"Generate new private key"** → **Generate key**
3. A JSON file will download — keep it safe

### Step 5: Set Environment Variables

Open the file `.env.local` in the project root and replace ALL placeholder values:

```env
# =============================================
# Firebase Client SDK (from Step 3)
# =============================================
NEXT_PUBLIC_FIREBASE_API_KEY=AIzaSy__YOUR_REAL_KEY__
NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN=firstframe-XXXXX.firebaseapp.com
NEXT_PUBLIC_FIREBASE_PROJECT_ID=firstframe-XXXXX
NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET=firstframe-XXXXX.appspot.com
NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID=123456789012
NEXT_PUBLIC_FIREBASE_APP_ID=1:123456789012:web:abcdef123456

# =============================================
# Firebase Admin SDK (from Step 4)
# Paste the ENTIRE JSON content on ONE line
# =============================================
FIREBASE_SERVICE_ACCOUNT_KEY={"type":"service_account","project_id":"firstframe-XXXXX","private_key_id":"...","private_key":"-----BEGIN PRIVATE KEY-----\n...\n-----END PRIVATE KEY-----\n","client_email":"firebase-adminsdk-...@firstframe-XXXXX.iam.gserviceaccount.com","client_id":"...","auth_uri":"https://accounts.google.com/o/oauth2/auth","token_uri":"https://oauth2.googleapis.com/token","auth_provider_x509_cert_url":"https://www.googleapis.com/oauth2/v1/certs","client_x509_cert_url":"..."}

# =============================================
# JWT Secret (generate a random strong string)
# =============================================
JWT_SECRET=change-this-to-a-random-64-char-string-in-production

# =============================================
# Resend Email API (optional for V1)
# Sign up at https://resend.com for free
# =============================================
RESEND_API_KEY=re_your_api_key
RESEND_FROM_EMAIL=onboarding@resend.dev

# =============================================
# App URL
# =============================================
NEXT_PUBLIC_APP_URL=http://localhost:3000
```

### Step 6: Create Your Admin Account

1. Go to Firebase Console → **Authentication** → **Users** tab
2. Click **"Add user"**
3. Enter your email & password (e.g. `admin@firstframe.in` / `Admin@123`)
4. Copy the **User UID** (shown after creation)

### Step 7: Seed the Database

Run the seed script to create the admin doc + sample data:

```bash
npm run seed
```

Or do it manually:
1. Go to Firebase Console → **Firestore Database**
2. Click **"Start collection"** → Collection ID: `admins`
3. Document ID: paste your **User UID** from Step 6
4. Add fields:
   - `email` (string): `admin@firstframe.in`
   - `name` (string): `Admin`
   - `createdAt` (timestamp): now

### Step 8: Run the App

```bash
npm run dev
```

Open [http://localhost:3000/admin/login](http://localhost:3000/admin/login) and login with your admin credentials.

## 🔐 Login Credentials Summary

| Dashboard | URL | Credentials |
|-----------|-----|-------------|
| Admin | `/admin/login` | Email + Password (Firebase Auth) |
| Brand | `/brand/login` | Company Name + Access Code (generated by Admin) |
| Creator | `/creator/join?token=xxx` | Magic link (generated by Admin) |

## 🔄 Typical Workflow

1. **Admin** logs in → adds Creators to pool → generates Access Code for Brand
2. **Brand** logs in with access code → browses Creators → invites them → adds Products
3. **Admin** approves invited creators → dispatches products
4. **Creator** joins via magic link → selects variant preference → reviews brief → submits content
5. **Brand** reviews submitted content → approves or requests revision
6. **Admin** monitors everything via Activity Log

## 🛠️ Troubleshooting

### "Firebase: No Firebase App" error
→ Your `.env.local` values are wrong or missing. Double-check they match Firebase Console exactly.

### Redirect loop on login pages
→ Clear browser cookies for localhost:3000. The middleware checks for session cookies.

### "Permission denied" in Firestore
→ Make sure your Firestore is in **test mode** for development.

### Blank white page
→ Check browser console (F12) for errors. Usually a missing env variable.
